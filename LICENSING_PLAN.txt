ACTIVATION RULES
-----------------
- 2 machine activations per license key (Gumroad tracks via uses count)
- First activation on a machine: call API with increment_uses_count=true, reject if uses > 2
- Monthly re-verify (every 30 days): call API with increment_uses_count=false (no use counted)
- Machine ID: generated UUID stored in Electron userData folder on first run
- License cache stored locally in Electron userData as JSON (key + lastVerified timestamp + machineId)


LICENSE STATUS STATES
----------------------
  "free"     — no license entered, all free-tier restrictions apply
  "pro"      — valid license, verified within 30 days, all features unlocked
  "overdue"  — license valid but >30 days since last check-in
                → red border around entire app window
                → funny joke popup appears
                → all pro features still work (grace mode) until verified
  "invalid"  — key was rejected or revoked by Gumroad

ARCHITECTURE
-------------
Main process (Node.js — NOT renderer, can't be bypassed via DevTools):
  electron/licenseManager.js
    - verifyWithGumroad(licenseKey, incrementUses) → calls Gumroad API via Node https
    - loadCache() / saveCache() → reads/writes userData/license.json
    - getMachineId() → reads/writes userData/machine.json (UUID)
    - getStatus() → returns current LicenseStatus object
    - activate(licenseKey) → first-time activation flow
    - periodicCheck() → called on app start, re-verifies if >30 days

IPC channels (preload.js exposes to renderer):
  license:get-status   → renderer asks for current status
  license:activate     → renderer sends key → main activates → returns result
  license:check        → renderer triggers manual re-verify

Renderer side:
  src/license/licenseStore.ts  — Zustand store: { isPro, status, checkOverdue }
  src/components/LicenseActivation.tsx — modal UI
  src/components/ProGate.tsx — reusable wrapper for locked features

Security level: MODERATE
  - Verification in main process (harder to bypass)
  - Light obfuscation of product ID in source
  - asar packaged (dist not plaintext browseable)
  - Determined users CAN crack it — user accepted this tradeoff
  - No online-only lockout (grace mode keeps app usable when offline)
